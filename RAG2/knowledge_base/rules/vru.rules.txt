[GENERAL]
- Output MUST be generated by filling the retrieved template(s); do NOT invent a new structure.
- Use user_config as the primary truth for values; fall back to scenario_details ONLY if explicitly required.
- Null values are VALID. Missing or null fields MUST NOT block generation.
- If a value is null and required for execution, apply a SAFE DEFAULT instead of failing.
- Units:
  - Speed: m/s in xosc. Convert kph to m/s by dividing by 3.6.
  - Distance: meters
  - Time: seconds
- Keep scenario naming stable: scenario_name MUST match JSON scenario_name.

[FAMILY IDENTIFICATION]
- Family MUST be determined from scenario_details.extra.adas_family == "VRU".
- If adas_family is not VRU, do NOT use VRU templates.
- VRU scenarios start with CP (pedestrian), CB (cyclist), or CM (motorcyclist).

[XOSC / SPEED RAMP]
- Hero and adversary/pedestrian accelerate in Story using SpeedAction:
  - dynamicsShape="linear" value="3.0" dynamicsDimension="time"
  - Triggered by SimulationTimeCondition value="1.0" rule="greaterThan"

[XOSC / PEDESTRIAN MOTION — CRITICAL]
- AcquirePositionAction is BROKEN for pedestrians in SR 0.9.16 — NEVER use it.
- RoutingAction is BROKEN for pedestrians in SR 0.9.16 — NEVER use it.
- Pedestrian motion: SpeedAction ONLY.
- Pedestrian heading at spawn controls walk direction.
- Pedestrian walk trigger: RelativeDistanceCondition longitudinal value="40.0" from hero.
- CPNA and CPLA appear visually identical in simulation — known SR 0.9.16 limitation.
  Pedestrian walks longitudinally as substitute for lateral crossing.

[XOSC / PEDESTRIAN CONTINUE GROUP]
- After hero brakes, pedestrian should continue walking.
- Add PedestrianContinueGroup ManeuverGroup triggered when hero SpeedCondition value="1.0" rule="lessThan".
- PedestrianContinueGroup SpeedAction: pedestrian continues at $pedestrianSpeed.

[XOSC / AEB BRAKING]
- AEB braking MUST be included as a separate ManeuverGroup on hero.
- AEB trigger for ALL VRU scenarios:
  - RelativeDistanceCondition entityRef="adversary" (or "pedestrian")
    relativeDistanceType="cartesianDistance" value="20.0" freespace="false" rule="lessThan"
- AEB brake action:
  - SpeedAction to 0.0 m/s
  - dynamicsShape="linear" value="3.0" dynamicsDimension="time"

[XOSC / LANE CHANGE (CMovertaking only)]
- NEVER use LaneChangeAction — causes segmentation fault in CARLA 0.9.15.
- Use FollowTrajectoryAction with Polyline vertices instead.
- FollowTrajectoryAction REQUIRES TimeReference child element:
  - <TimeReference><Timing domainAbsoluteRelative="absolute" scale="1.0" offset="0.0"/></TimeReference>
- Polyline MUST have 4 or more vertices or SR throws IndexError.

[XOSC / TRIGGERS]
- SimulationTimeCondition value MUST always be "1.0" (NEVER "0.1" or "0.0").
- Act StartTrigger: SimulationTimeCondition value="1.0" rule="greaterThan"
- Speed ramp StartTrigger: SimulationTimeCondition value="1.0" rule="greaterThan"
- WaitGroup StartTrigger: SimulationTimeCondition value="55.0" rule="greaterThan"

[XOSC / WAIT GROUP]
- ALWAYS add a WaitGroup ManeuverGroup as the last ManeuverGroup in the Act.
- WaitGroup keeps the Act alive until the 60s timeout fires.
- Without WaitGroup, SR exits early when all ManeuverGroups complete.
- WaitGroup triggers at SimulationTimeCondition value="55.0".
- WaitGroup action: SpeedAction setting adversary/pedestrian to 0.0.

[XOSC / STORYBOARD STRUCTURE]
- VRU pedestrian scenarios MUST have 5 ManeuverGroups in this order:
  1. HeroAccelGroup — hero accelerates to $heroSpeed
  2. PedestrianWalkGroup — pedestrian starts walking at $pedestrianSpeed
  3. HeroAEBGroup — hero brakes when cartesianDistance < 20.0
  4. PedestrianContinueGroup — pedestrian continues after hero stops
  5. WaitGroup — keeps Act alive at t=55s

- VRU cyclist/motorcyclist scenarios MUST have 4 ManeuverGroups:
  1. HeroAccelGroup — hero accelerates to $heroSpeed
  2. AdversaryAccelGroup — adversary accelerates to $adversarySpeed
  3. HeroAEBGroup — hero brakes when cartesianDistance < 20.0
  4. WaitGroup — keeps Act alive at t=55s

[XOSC / STOP]
- ALWAYS include StopTrigger in Act: SimulationTimeCondition value="60.0" rule="greaterThan"
- Global StopTrigger MUST use criteria_CollisionTest ParameterCondition pattern:
  - <Condition name="criteria_CollisionTest" delay="0.0" conditionEdge="rising">
      <ByValueCondition>
        <ParameterCondition parameterRef="criteria_CollisionTest" value="" rule="lessThan"/>
      </ByValueCondition>
    </Condition>
- Do NOT include criteria_DrivenDistanceTest.
- parameterRef MUST be "criteria_CollisionTest" (not empty string).
- Without criteria_ ParameterCondition in global StopTrigger, SR exits immediately with
  "Nothing to analyze" error.

[KNOWN LIMITATIONS — SR 0.9.16]
- CPRA/CPRC (reverse driving): SR does not support reverse vehicle motion via xosc.
  SpeedAction always drives forward. Tool generates xosc but scenario will not work correctly.
- CPTA-50 (pedestrian turning at junction): Complex junction geometry not implemented.
- CBTA-50 (cyclist turning at junction): Complex junction geometry not implemented.
- CPNA and CPLA visually identical: Pedestrian lateral crossing not supported in SR 0.9.16.
  Both simulate longitudinal pedestrian walking as substitute.

# ================================
# LSS RULE PACK (v1)
# Family: LSS
# Purpose: Hard constraints for code generation (Python + XOSC)
# ================================

[GENERAL]
- Output MUST be generated by filling the retrieved template(s); do NOT invent a new structure.
- Use user_config as the primary truth for values; only fall back to scenario_details if a field is missing.
- Do NOT hallucinate values. If a required field is missing, keep the placeholder unchanged and add a TODO comment near it.
- Units:
  - Speed: user_config typically in kph; convert to m/s only where template expects m/s placeholders.
  - Distance: meters
  - Time: seconds
- Keep scenario naming stable: scenario_name MUST match JSON scenario_name.
- Do not introduce hardcoded filepaths, map names, or external assets unless explicitly provided.

[FAMILY IDENTIFICATION]
- Family MUST be determined from scenario_details.extra.adas_family == "LSS".
- If adas_family is not LSS, do NOT use LSS templates.

[PYTHON / SCENARIORUNNER]
- Must produce ScenarioRunner-compatible code (BasicScenario subclass).
- Do not change the BasicScenario class signature used in the retrieved template.
- Do not add new imports unless the template already includes them.
- Ego vehicle is provided by ScenarioRunner (ego_vehicles[0]); do NOT spawn ego unless the template explicitly does.
- "other_actors" MUST include the target vehicle spawned by the scenario.
- Target spawning:
  - Use blueprint from target_blueprint placeholder.
  - If spawn fails, raise RuntimeError with a short message.
- Trigger logic:
  - Prefer TTC trigger if ttc_trigger_s is present and non-null.
  - Otherwise use trigger_distance_m.
  - Do not use both simultaneously.
- Behavior tree requirements:
  - Must be deterministic (no random behavior).
  - Must include a post-trigger duration (post_trigger_duration_s).
- Lane change behavior (Python template):
  - Lane change direction MUST come from lane_change_direction ("left" or "right").
  - lane_change_duration_s and lane_change_distance_m MUST be numeric and non-negative.
  - Do not invent alternate lane-change implementations unless retrieved as an atom.
  - If lane-change is not required by the scenario, set lane_change_distance_m = 0 and lane_change_duration_s = 0 (or keep placeholders with TODO).
- Criteria:
  - Include CollisionTest for ego vehicle unless criteria_enable is false.
- Never output markdown fences. Output must be raw code inside the JSON response field.

[XOSC]
- Must be valid OpenSCENARIO structure:
  - <OpenSCENARIO> root present.
  - Use <Storyboard> with at least one <Story>, <Act>, <ManeuverGroup>, <Maneuver>, <Event>, <Action>.
- Entity naming:
  - Use stable entity names: "ego" and "target" unless template uses different.
  - All references must match declared entities exactly.
- Speed actions:
  - Use AbsoluteTargetSpeed with explicit value.
- Triggers:
  - Use either TTC-based trigger or distance-based trigger, consistent with JSON fields.
  - If both nodes exist in template, fill the unused one with non-firing values (e.g., huge TTC or very large distance).
- Lane change action (XOSC template):
  - lane_change_offset MUST be consistent with lane_change_direction:
    - if direction is "left" -> lane_change_offset should be +1 (typical)
    - if direction is "right" -> lane_change_offset should be -1 (typical)
  - lane_change_dynamics_shape / value / dimension must be filled with valid values (no empty strings).
  - Do not use invalid tags or schema variants; stick to the template.
- No placeholder map/road file paths unless provided.

[PLACEHOLDER FILLING / MAPPING]
- Fill placeholders only from the minimal JSON snippet fields.
- If field names differ, map them explicitly (e.g., user_config.lss.ego_speed_kph -> ego_speed_kph).
- Ensure numeric placeholders are valid floats/ints in the output (no quotes around numbers).
- Ensure string placeholders are quoted in output (e.g., blueprint IDs).
- Keep transforms consistent:
  - Use world coordinates exactly as provided.
  - Do not guess spawn points or lane indexes.

[SAFETY / CONSISTENCY]
- Do not add traffic, pedestrians, weather, friction, sensor settings unless explicitly specified.
- Keep logic minimal: spawn, trigger, motion, lane-change, criteria, timeout.

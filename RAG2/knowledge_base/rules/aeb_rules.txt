# ================================
# AEB RULE PACK (v2 – null-friendly)
# Family: AEB
# Purpose: Guidance for code generation (Python + XOSC)
# ================================

[GENERAL]
- Output MUST be generated by filling the retrieved template(s); do NOT invent a new structure.
- Use user_config as the primary truth for values; fall back to scenario_details ONLY if explicitly required.
- Null values are VALID. Missing or null fields MUST NOT block generation.
- If a value is null and required for execution, apply a SAFE DEFAULT instead of failing.
- Units:
  - Speed: user_config typically in kph; convert to m/s only when required by template.
  - Distance: meters
  - Time: seconds
  - Deceleration: m/s^2 (positive magnitude).
- Keep scenario naming stable: scenario_name MUST match JSON scenario_name.
- Do not introduce hardcoded filepaths, map names, or external assets unless explicitly provided.

[FAMILY IDENTIFICATION]
- Family MUST be determined from scenario_details.extra.adas_family == "AEB".
- If adas_family is not AEB, do NOT use AEB templates.

[PYTHON / SCENARIORUNNER]
- Must produce ScenarioRunner-compatible code (BasicScenario subclass).
- Do not change the BasicScenario class signature used in the retrieved template.
- Do not add new imports unless the template already includes them.
- Ego vehicle is provided by ScenarioRunner (ego_vehicles[0]); do NOT spawn ego unless the template explicitly does.
- "other_actors" MUST include the target vehicle spawned by the scenario.

[PYTHON / TRIGGERS]
- Trigger behavior MUST follow user_config.trigger.type:
  - START_IMMEDIATELY:
    - No TTC or distance trigger required.
    - Scenario starts at simulation time > 0.
  - TTC:
    - Use TTC trigger if ttc_s is provided.
    - If ttc_s is null, apply a safe default (e.g., 2.0s).
  - DISTANCE:
    - Use distance trigger if distance_m is provided.
    - If distance_m is null, apply a safe default (e.g., 40m).
- Do NOT require TTC or distance unless the trigger type explicitly demands it.
- Do NOT activate TTC and distance triggers simultaneously.

[PYTHON / BEHAVIOR]
- Behavior tree MUST be deterministic.
- Target motion:
  - target_speed_kph == 0 → stationary target.
  - target_speed_kph > 0 → moving target.
  - If braking fields are provided → include braking logic.
  - If braking fields are null → do NOT invent braking.
- Always include a post-trigger runtime so interaction completes.
- Criteria:
  - Include CollisionTest unless explicitly disabled.

[XOSC]
- Must be valid OpenSCENARIO structure.
- ALWAYS include:
  - <Entities>
  - <Storyboard> with <Story>, <Act>, <ManeuverGroup>, <Maneuver>, <Event>
- Entity naming:
  - Use stable names: "ego", "target", "vru" (if present).
- Init phase MUST include:
  - TeleportAction + Position (LanePosition or WorldPosition) for each entity.
  - AbsoluteTargetSpeed for each entity.
- Placement rules:
  - Use initial_gap_m / initial_distance_m if provided.
  - If both are null → default to a safe value (e.g., 15m).
  - lateral_offset_m defaults to 0 if null.
  - overlap_percent is OPTIONAL; ignore if null.

[XOSC / TRIGGERS]
- Trigger must match user_config.trigger.type:
  - START_IMMEDIATELY → SimulationTimeCondition.
  - TTC → TimeToCollisionCondition.
  - DISTANCE → RelativeDistanceCondition.
- If trigger parameters are null, apply safe defaults instead of failing.
- Do NOT activate multiple trigger types simultaneously.

[XOSC / STOP]
- ALWAYS include StopTrigger.
- If timeout_s is null → default to 60s.

[SAFETY / CONSISTENCY]
- Do not add traffic, pedestrians, weather, sensors, friction unless explicitly specified.
- Keep logic minimal: spawn, placement, trigger, behavior, criteria, stop.
- Prefer execution-safe defaults over TODOs or placeholders.

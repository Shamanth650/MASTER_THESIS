# ================================
# AEB RULE PACK (v4 – CARLA ScenarioRunner VALIDATED)
# Family: AEB
# Purpose: Guidance for code generation (Python + XOSC)
# Validated: CARLA 0.9.15 / ScenarioRunner 0.9.16 / Town01
# ================================

[GENERAL]
- Output MUST be generated by filling the retrieved template(s); do NOT invent a new structure.
- Use user_config as the primary truth for values; fall back to scenario_details ONLY if explicitly required.
- Null values are VALID. Missing or null fields MUST NOT block generation.
- If a value is null and required for execution, apply a SAFE DEFAULT instead of failing.
- Units:
  - Speed: user_config typically in kph; convert to m/s only when required by template.
  - Distance: meters
  - Time: seconds
  - Deceleration: m/s^2 (positive magnitude).
- Keep scenario naming stable: scenario_name MUST match JSON scenario_name.

[FAMILY IDENTIFICATION]
- Family MUST be determined from scenario_details.extra.adas_family == "AEB".
- If adas_family is not AEB, do NOT use AEB templates.

[PYTHON / SCENARIORUNNER]
- Must produce ScenarioRunner-compatible code (BasicScenario subclass).
- Do not change the BasicScenario class signature used in the retrieved template.
- Do not add new imports unless the template already includes them.
- Ego vehicle is provided by ScenarioRunner (ego_vehicles[0]); do NOT spawn ego unless the template explicitly does.
- "other_actors" MUST include the target vehicle spawned by the scenario.

[PYTHON / TRIGGERS]
- Trigger behavior MUST follow user_config.trigger.type:
  - START_IMMEDIATELY:
    - No TTC or distance trigger required.
    - Scenario starts at simulation time > 0.
  - TTC:
    - Use TTC trigger if ttc_s is provided.
    - If ttc_s is null, apply a safe default (e.g., 2.0s).
  - DISTANCE:
    - Use distance trigger if distance_m is provided.
    - If distance_m is null, apply a safe default (e.g., 40m).
- Do NOT require TTC or distance unless the trigger type explicitly demands it.
- Do NOT activate TTC and distance triggers simultaneously.

[PYTHON / BEHAVIOR]
- Behavior tree MUST be deterministic.
- Target motion:
  - target_speed_kph == 0 → stationary target.
  - target_speed_kph > 0 → moving target.
  - If braking fields are provided → include braking logic.
  - If braking fields are null → do NOT invent braking.
- Always include a post-trigger runtime so interaction completes.
- Criteria:
  - Include CollisionTest unless explicitly disabled.

[XOSC]
- Must be valid OpenSCENARIO 1.x structure executable in CARLA ScenarioRunner.
- ALWAYS include:
  - <Entities>
  - <RoadNetwork> with BOTH <LogicFile filepath="Town01"/> AND <SceneGraphFile filepath=""/>
  - <Storyboard> with <Story>, <Act>, <ManeuverGroup>, <Maneuver>, <Event>
  - <GlobalAction><EnvironmentAction> block in Init with weather/sun/fog/precipitation
- Entity naming:
  - Use stable names: "hero" for ego vehicle, "adversary" for target vehicle.
- Vehicle blueprints:
  - hero MUST use: vehicle.lincoln.mkz_2017 (underscore before 2017, not mkz2017)
  - adversary MUST use: vehicle.tesla.model3
- Vehicle properties MUST include both:
  - <Property name="type" value="ego_vehicle"/> for hero
  - <Property name="role_name" value="hero"/> for hero
  - <Property name="type" value="simulation"/> for adversary
  - <Property name="role_name" value="adversary"/> for adversary
- Performance values MUST be:
  - maxSpeed="69.444" maxAcceleration="200" maxDeceleration="10.0"
- FileHeader date MUST be full datetime format: date="2020-03-20T12:00:00"

[XOSC / SPAWN POSITIONS — CRITICAL]
- ALWAYS use LanePosition for ALL entities. NEVER use WorldPosition or RelativeRoadPosition.
- WorldPosition causes river/off-road spawns in Town01. RelativeRoadPosition causes off-road spawns.
- LanePosition is the ONLY spawn method validated to work in CARLA 0.9.15 Town01.

- CCR scenarios (CCRs, CCRm, CCRb) — both on same straight road:
  - hero:     LanePosition roadId="12" laneId="-1" s="156.84" offset="0.0"
  - adversary: LanePosition roadId="12" laneId="-1" s="193.66" offset="0.0"

- CCFtap / CCFtab — perpendicular roads (junction):
  - hero:     LanePosition roadId="4"  laneId="-1" s="197.98" offset="0.0"
  - adversary: LanePosition roadId="12" laneId="-1" s="193.66" offset="0.0"

- CCFhos (head-on straight) — opposite lanes:
  - hero:     LanePosition roadId="12" laneId="-1" s="156.84" offset="0.0"
  - adversary: LanePosition roadId="12" laneId="1"  s="193.66" offset="0.0"

- CCFhol (head-on lane change) — adversary starts opposite lane, changes into hero lane:
  - hero:     LanePosition roadId="12" laneId="-1" s="156.84" offset="0.0"
  - adversary: LanePosition roadId="12" laneId="1"  s="193.66" offset="0.0"
  - adversary uses FollowTrajectoryAction (NOT LaneChangeAction — causes segfault in CARLA 0.9.15)

- Init speed MUST be 0.0 for ALL entities in Init phase.
- Speed ramp MUST only happen in Story/Act ManeuverGroup, NEVER in Init.

[XOSC / SPEED RAMP]
- Hero and adversary accelerate in Story using SpeedAction:
  - dynamicsShape="linear" value="3.0" dynamicsDimension="time"
  - Triggered by SimulationTimeCondition value="1.0" rule="greaterThan"
- Speed values per scenario (m/s):
  - CCRs:  hero=8.0,    adversary=0.0
  - CCRm:  hero=8.0,    adversary=3.0
  - CCRb:  hero=8.0,    adversary=8.0 (adversary brakes after)
  - CCFtap: hero=10.0,  adversary=3.0
  - CCFtab: hero=10.0,  adversary=3.0
  - CCFhos: hero=8.333, adversary=8.333
  - CCFhol: hero=8.333, adversary=8.333

[XOSC / AEB BRAKING]
- AEB braking MUST be included as a separate ManeuverGroup on hero.
- CCR scenarios — AEB trigger:
  - RelativeDistanceCondition entityRef="adversary" relativeDistanceType="cartesianDistance"
    value="12.0" freespace="false" rule="lessThan"
- CCF/head-on scenarios — AEB trigger:
  - RelativeDistanceCondition entityRef="adversary" relativeDistanceType="cartesianDistance"
    value="20.0" freespace="false" rule="lessThan"
- AEB brake action (ALL scenarios):
  - SpeedAction to 0.0 m/s
  - dynamicsShape="linear" value="3.0" dynamicsDimension="time"

[XOSC / LANE CHANGE (CCFhol only)]
- NEVER use LaneChangeAction — causes segmentation fault in CARLA 0.9.15.
- Use FollowTrajectoryAction with Polyline vertices instead.
- FollowTrajectoryAction REQUIRES TimeReference child element:
  - <TimeReference><Timing domainAbsoluteRelative="absolute" scale="1.0" offset="0.0"/></TimeReference>
- Polyline MUST have 4 or more vertices or SR throws IndexError.
- Example vertices for lane change (laneId=1 → laneId=-1):
  - Vertex time="0.0": LanePosition roadId="12" laneId="1"  s="193.66" offset="0.0"
  - Vertex time="1.5": LanePosition roadId="12" laneId="1"  s="188.0"  offset="-1.75"
  - Vertex time="3.0": LanePosition roadId="12" laneId="-1" s="182.0"  offset="0.0"
  - Vertex time="10.0": LanePosition roadId="12" laneId="-1" s="170.0" offset="0.0"
- Both hero AND adversary need separate AEB ManeuverGroups in CCFhol.
- Lane change triggers when adversary is 35m cartesianDistance from hero.

[XOSC / TRIGGERS]
- SimulationTimeCondition value MUST always be "1.0" (NEVER "0.1" or "0.0").
- Act StartTrigger: SimulationTimeCondition value="1.0" rule="greaterThan"
- Speed ramp StartTrigger: SimulationTimeCondition value="1.0" rule="greaterThan"
- WaitGroup StartTrigger: SimulationTimeCondition value="55.0" rule="greaterThan"

[XOSC / WAIT GROUP]
- ALWAYS add a WaitGroup ManeuverGroup as the last ManeuverGroup in the Act.
- WaitGroup keeps the Act alive until the 60s timeout fires.
- Without WaitGroup, SR exits early when all ManeuverGroups complete.
- WaitGroup triggers at SimulationTimeCondition value="55.0".
- WaitGroup action: SpeedAction keeping adversary at its current speed.

[XOSC / STOP]
- ALWAYS include StopTrigger in Act: SimulationTimeCondition value="60.0" rule="greaterThan"
- Global StopTrigger MUST use criteria_CollisionTest ParameterCondition pattern:
  - <Condition name="criteria_CollisionTest" delay="0.0" conditionEdge="rising">
      <ByValueCondition>
        <ParameterCondition parameterRef="criteria_CollisionTest" value="" rule="lessThan"/>
      </ByValueCondition>
    </Condition>
- Do NOT include criteria_DrivenDistanceTest for VRU or short-distance scenarios.
- criteria_CollisionTest parameterRef MUST be "criteria_CollisionTest" (not empty, not "distance_success").
- Without criteria_ ParameterCondition in global StopTrigger, SR exits immediately with
  "Nothing to analyze" error.

[SAFETY / CONSISTENCY]
- ALWAYS include EnvironmentAction with weather in Init GlobalAction.
- Do not add traffic, pedestrians, sensors, friction unless explicitly specified.
- Keep logic minimal: spawn, placement, trigger, behavior, AEB brake, WaitGroup, criteria, stop.
- Prefer execution-safe defaults over TODOs or placeholders.
- roadId values MUST be non-zero integers. laneId MUST be negative for forward travel (-1).
